# اسکنر کلادفلر

> [English](./README.md) | [فارسی](./README_FA.md)

**cf-scanner** یک اسکنر رنج ایپی چند نخی و کاربرپسند است.
به سرعت بررسی میکند که کدام میزبانها در یک رنج فعال هستند و آیا سرویسهای HTTP (پورت 80) و HTTPS (پورت 443) قابل دسترسی است، و همه چیز را در یک جدول مرتب در ترمینال نمایش میدهد.

---

## ویژگی ها

- **پینگ ICMP برای شناسایی میزبانهای زنده**
- **بررسی پورت TCP 80 (HTTP) و پورت TCP 443 (HTTPS)**
- **جدول مرتب ترمینال با شماره گذاری صحیح ردیفها (1, 2, 3, …)**
- **ذخیره ایپیهای فعال در `actives.txt`**
- **چند نخی برای اسکن سریعتر**
- **شامل فایل آماده `hosts.txt` با تمام رنجهای ایپی کلادفلر**
  → می‌توانید تمام ایپیهای کلادفلر را فقط با وارد کردن نام این فایل اسکن کنید

## ویژه: اسکن تمام ایپیهای کلادفلر

پروژه شامل فایل **`hosts.txt`** است که حاوی **تمام رنجهای ایپی کلادفلر** است.

برای اسکن تمام ایپیهای کلادفلر:

1. مطمئن شوید `hosts.txt` در همان دایرکتوری `scanner.py` قرار دارد.
2. وقتی اسکنر نام فایل رنج را می‌پرسد، به سادگی تایپ کنید:

```text
hosts.txt
```

اسکنر تمام ایپیهای کلادفلر را از `hosts.txt` بررسی می‌کند و:

- نتیجه را در جدول نمایش می‌دهد،
- تمام ایپیهای زنده را در `actives.txt` ذخیره می‌کند.

## نصب: استفاده از گیت

این روش برای زمانی است که کد را در یک مخزن گیت (GitHub, GitLab و غیره) میزبانی می‌کنید.

### 1. کلون مخزن

```bash
git clone https://github.com/hu-matin/cf-scanner.git
cd cf-scanner
```

### 2. نصب وابستگی‌ها

اگر `requirements.txt` دارید:

```bash
pip install -r requirements.txt
```

### 3. آماده‌سازی فایل‌های رنج

**گزینه A – رنجهای خودتان**

یک فایل متنی ایجاد کنید، مثلاً `ranges.txt`، و یک رنج در هر خط قرار دهید.
در این نسخه ساده، هر خط فرض می‌شود یک بلوک `/24` است:

```text
3.25.205.0/24
10.0.0.0/24
```

**گزینه B – کلادفلر**

فقط از فایل شامل شده **`hosts.txt`** استفاده کنید – کار اضافی لازم نیست.

### 4. اجرای اسکنر

```bash
python scanner.py
```

سپس:

- برای اسکن رنجهای خودتان، وارد کنید: `ranges.txt`
- برای اسکن ایپیهای کلادفلر، وارد کنید: `hosts.txt`
- تعداد نخ را انتخاب کنید (مثلاً 50، یا Enter برای پیش‌فرض)

نمونه خروجی:

```text
[#]  IP address          PING   P80   P443  DOMAIN
────────────────────────────────────────────────────────
[ 1]  3.25.205.10        OK     OPEN  CLOSE example.com
[ 2]  3.25.205.11        FAIL   CLOSE CLOSE no-domain
...
```

تمام میزبانهای زنده نیز در `actives.txt` ذخیره خواهند شد.

## روش 2: استفاده از ZIP (بدون گیت)

اگر پروژه را به عنوان یک **فایل ZIP** دریافت کرده‌اید (از تلگرام، وب و غیره):

### 1. استخراج ZIP

- ZIP را (مثلاً `cf-scanner.zip`) جایی قرار دهید (مثل دسکتاپ)
- راست کلیک → **Extract All...** (یا از WinRAR / 7-Zip استفاده کنید)
- وارد پوشه استخراج شده شوید – باید `scanner.py`، `hosts.txt` و `modules/` را ببینید.

### 2. باز کردن CMD / PowerShell در آن پوشه

در ویندوز:

- روی نوار مسیر در Explorer کلیک کنید، `cmd` تایپ کنید، Enter بزنید
  یا
- Shift + راست کلیک → **Open PowerShell window here**

### 3. نصب وابستگی‌ها

با `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 4. انتخاب آنچه را اسکن کنید

- برای رنجهای ایپی خودتان: `ranges.txt` ایجاد کنید و رنجهای خود را اضافه کنید (یک در هر خط).
- برای کلادفلر: به سادگی از `hosts.txt` ارائه شده استفاده کنید.

### 5. اجرای اسکنر

```bash
python scanner.py
```

سپس:

- برای اسکن کلادفلر → وارد کنید: `hosts.txt`
- برای رنجهای خودتان → وارد کنید: `ranges.txt` (یا هر نام فایلی که انتخاب کردید)
- تعداد نخ را تنظیم کنید

جدول زنده و مرتب را در ترمینال خواهید دید، و `actives.txt` شامل تمام میزبانهای پاسخ‌دهنده خواهد بود.

## نکات

- اسکن کندتر اما پایدارتر می‌خواهید؟ **از نخهای کمتر استفاده کنید** (مثلاً 10).
- نیاز به اسکن سریع‌تر در شبکه قوی دارید؟ تعداد نخ را افزایش دهید، اما بار شبکه خود را مراقب باشید.
- می‌توانید `actives.txt` را برای ابزارهای دیگر یا تحلیل بیشتر دوباره استفاده کنید.
- برای بررسی سریع سلامت کلادفلر، همیشه **`hosts.txt`** را به عنوان فایل رنج استفاده کنید.
